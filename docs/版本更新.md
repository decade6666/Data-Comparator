## 1.6
发布日期：2025-08-11

### 新增
- 模块化重构：拆分为 `src/gui`、`src/core`、`src/utils`，可维护性与可测性提升
- 参数多配置管理：
  - 支持新建/删除/重命名/导入/导出配置
  - 配置持久化到 AppData：`PyDataCompare/temp/configs/`
- 更安全的文件处理：
  - 预处理阶段另存 `_nofilter_` 副本到 AppData，保护原文件
  - 自动清理筛选器；按需删除 `exclude_sheets`
  - 检测并尝试移除 Zone.Identifier（互联网受保护来源）
- 高亮与表头增强：
  - 表头新增/删除列以不同颜色标记
  - 更新行仅按变更单元格高亮；新增/删除行整行高亮
- 多线程与性能：
  - 默认最大线程数为 CPU-1，可在“高级设置”中修改
  - 条件 GC 与内存峰值保护，降低大文件处理风险
- UI 与交互：
  - 统一浅色主题（可选深色模式）
  - 进度区优化，支持停止操作
  - 图标资源加载兼容开发/打包模式

### 变更
- 将单文件脚本（3,800+ 行）拆解为多个职责单一的模块与类
- `logger` 简化为通用转发函数；日志以回调方式传入 GUI
- `read_single_sheet_from_excel`：
  - 仅按需读取目标 Sheet，读取完成即关闭文件
  - 兼容缺失/空表头，保留列名/标签到 `DataFrame.attrs`
- `perform_full_comparison`：
  - 使用 `_ANCHOR` 外连接合并，旧列使用 `_OLD_` 后缀
  - 输出差异明细 `diff_dict` 便于精准高亮
- 统一“更新情况（标记）”列置于第 1 列

### 修复
- 修复多处空表/空列/重复列名导致的异常
- 修复高亮时列索引错位问题（以 `sas_file_names` 构建映射）
- 修复受保护文件导致的读取失败路径，增加回退清理策略

### 移除/弃用
- 旧版内嵌 GUI 更新/进度类被替换为 `gui_update_manager.py`、`progress_manager.py`
- 大表拆分配置暂未启用（保留为后续扩展方向）

### 兼容性
- Python ≥ 3.8
- Windows 10+
- 建议使用 .xlsx/.xlsm；旧 `.xls` 建议转换

### 升级指南
- 业务使用：界面操作与结果含义保持一致，可直接迁移
- 自动化部署/打包：请使用 `pyproject.toml` 或 `build/app.spec` 更新构建
- 二次开发：参考 `docs/开发文档.md` 的扩展点，避免直接修改核心比对逻辑
