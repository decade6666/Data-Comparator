# 开发文档

本文档面向研发/维护者，介绍架构、关键模块、数据流、扩展点、构建发布与测试要点。

## 架构概览
- GUI 层（`src/gui`）：Tkinter + ttkbootstrap
  - `main_window.py`：主界面、事件协调、进度/日志、参数面板
  - `parameter_manager.py`：配置管理（AppData 下 `configs` 子目录）
  - `components/`、`dialogs/`：UI 组件与弹窗
- Core 层（`src/core`）：Excel 读取、差异检测、写回高亮
  - `data_comparison.py`：流程编排、并发、多表/单表处理、差异聚合
  - `excel_header_utils.py`：头部行（SASFieldLabel）/锚点行（SASFieldName）读取
  - `excel_utils.py`、`highlight_utils.py`：表头替换、Sheet/单元格高亮
- Utils 层（`src/utils`）：通用工具
  - `file_utils.py`：受保护文件处理、移除筛选器、AppData 路径、缓存清理、资源加载
  - `config_manager.py`：运行期配置聚合（颜色、线程、合并删除数据等）
  - `gui_update_manager.py`、`progress_manager.py`、`logger.py`、`stop_flag_utils.py`

依赖管理使用 `pyproject.toml`，可选 extras：`performance`、`build`、`dev`。

## 运行入口
- `src/main.py`：创建 ttk.Window、设置图标与主题，初始化 `DatasetComparatorGUI`
- `run.py`：保证项目根在 `sys.path`，便于 `python run.py` 直接运行

## 配置与持久化
- 参数由 `ParameterManager` 管理，存于 AppData：`PyDataCompare/temp/configs/<配置名>.json`
- 关键参数结构（示例）：
```json
{
  "old_file_path": "",
  "new_file_path": "",
  "output_directory": "",
  "anchor_row_num": 1,
  "header_row_num": 1,
  "max_workers": 7,
  "merge_deleted_data": true,
  "common_cols": [],
  "exclude_sheets": [],
  "default_keys": [],
  "sheet_key_map": {},
  "colors": {
    "highlight_fill": "#FFE5E5",
    "missing_sheet_tab": "#DC143C",
    "new_sheet_tab": "#00FF00"
  }
}
```
- 运行期整合到 `ConfigManager.update_from_parameters`，并将颜色转为 openpyxl `PatternFill`

## 数据流与核心流程
- GUI 收集参数 → 初始化 `ThreadSafeProgressManager`/`ConfigManager`
- 预处理：
  - `check_and_remove_file_protection`：复制到 AppData/temp，移除 Zone.Identifier，清理 AutoFilter，按需删除 `exclude_sheets`
  - `validate_excel_file`：多引擎快速验读
- 逐 Sheet 流程（线程池执行）：
  - `read_single_sheet_from_excel`：一次性只读目标 Sheet；解析 `SASFieldLabel/Name`，生成 `attrs`（列名/标签/映射）
  - 缺失/新增分支：`process_missing_sheet` / `process_new_sheet`
  - 正常比对：
    - `create_anchor_by_sas_names` 生成 `_ANCHOR`
    - 外连接合并 `_ANCHOR`，旧列后缀 `_OLD_`
    - `compare_columns_by_sas_names` 产出单元格 diff 明细
    - 汇总“新增/删除/更新”并写入“更新情况（标记）”列
  - 写回：使用 `excel_utils.apply_highlight_to_worksheet` 进行表头/单元格/整行高亮
- 资源回收：条件 GC `_maybe_gc_collect`，并在 finally 更新 Sheet 状态

## 线程与停止机制
- 线程池大小：`ConfigManager.max_workers`，默认 `max(1, cpu_count()-1)`
- 停止：`threading.Event` + `stop_flag_utils.check_stop_frequently` 在关键循环中主动抛出 `InterruptedError`
- 进度：`ThreadSafeProgressManager.update_sheet_progress` 更新 GUI

## 扩展点
- 比对策略：在 `perform_full_comparison` 中扩展匹配与差异规则
- 表头解析：`read_single_sheet_from_excel` 可支持更多头部布局或多行表头合并
- 高亮策略：`excel_utils.apply_highlight_to_worksheet` 可扩展色彩/样式规则
- 导入导出：在 `ParameterManager` 增加 JSON Schema 校验与版本迁移
- 大表拆分：`BigSheetSplitConfig`（旧版参考）可逐步引入更细粒度分块

## 构建与发布
- 开发安装：
  ```bash
  pip install -e .[performance]
  ```
- PyInstaller：
  - 直接命令（示例见 docs/README.md）
  - 使用 `build/app.spec` 或 `pyproject.tool.pyinstaller` 配置
- 可执行产物包含 `assets/icons/app_icon.ico` 等资源，注意 `datas` 与相对路径处理（`get_resource_path`）

## 日志与临时文件
- 控制台打印 + GUI 回调（转发至 `progress_manager.safe_log`）
- `_nofilter_`、`excel_extract_*` 均位于 AppData/temp；提供 `cleanup_nofilter_files` 清理

## 测试要点
- 大小混合：空 Sheet、有合并单元格、非标准表头、长列名、重复列名
- 受保护来源：Zone.Identifier 删除失败的回退行为
- 性能回归：线程数变化、超大 Sheet、diff 覆盖路径
- 颜色正确性：表头新增/删除标识、更新单元格高亮

## 已知限制/注意
- `xlrd` 不支持 .xlsx；尽量使用 openpyxl
- 非结构化表头需手动指定表头/锚点行
- 深色模式在部分 Windows 主题不可用
